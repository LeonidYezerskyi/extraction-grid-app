# Деплой на Streamlit Cloud

## Важливо! 
Streamlit Cloud **НЕ** деплоїться через локальний інтерфейс. Потрібно використовувати веб-інтерфейс на share.streamlit.io.

## Крок 1: Підготовка репозиторію на GitHub

1. Переконайтеся, що ваш проект завантажений на GitHub:
   - Репозиторій має бути **публічним** (або ви маєте Streamlit Cloud Team/Enterprise)
   - Всі файли мають бути закомічені та запушені

2. Перевірте, що є файл `requirements.txt` з усіма залежностями (вже є ✅)

3. Переконайтеся, що головний файл називається `app.py` (вже є ✅)

## Крок 2: Деплой через веб-інтерфейс Streamlit Cloud

1. **Відкрийте браузер** і перейдіть на: https://share.streamlit.io/

2. **Увійдіть через GitHub**:
   - Натисніть "Sign in" або "Get started"
   - Авторизуйтеся через ваш GitHub акаунт
   - Дозвольте Streamlit Cloud доступ до ваших репозиторіїв

3. **Створіть новий додаток**:
   - Натисніть "New app" або "Deploy an app"
   - Виберіть ваш GitHub репозиторій зі списку
   - Виберіть гілку (зазвичай `main` або `master`)

4. **Налаштуйте деплой**:
   - **Main file path**: `app.py`
   - **Python version**: залиште за замовчуванням (або виберіть 3.11+)
   - **App URL**: можна залишити автоматичний або вказати свій

5. **Натисніть "Deploy"**

## Крок 3: Очікування деплою

- Streamlit Cloud автоматично встановить залежності з `requirements.txt`
- Деплой займе 2-5 хвилин
- Ви побачите логи в реальному часі

## Крок 4: Перевірка

Після успішного деплою ви отримаєте URL типу:
```
https://your-app-name.streamlit.app
```

## Можливі проблеми та рішення

### Проблема: "Repository not found"
**Рішення:**
- Переконайтеся, що репозиторій публічний
- Перевірте, що ви авторизувалися правильним GitHub акаунтом
- Переконайтеся, що репозиторій існує і доступний

### Проблема: "Module not found"
**Рішення:**
- Перевірте, що всі залежності є в `requirements.txt`
- Переконайтеся, що всі локальні модулі (ingest.py, normalize.py тощо) закомічені в репозиторій

### Проблема: "App failed to start"
**Рішення:**
- Перевірте логи деплою на сторінці Streamlit Cloud
- Переконайтеся, що `app.py` має функцію `main()` та `if __name__ == "__main__": main()`
- Перевірте, що немає помилок синтаксису

## Додаткові налаштування (опціонально)

Можна створити файл `.streamlit/config.toml` для додаткової конфігурації:

```toml
[server]
headless = true
enableCORS = false
enableXsrfProtection = true

[browser]
gatherUsageStats = false
```

Але це не обов'язково для базового деплою.

## Оновлення додатку

Після кожного push до GitHub, Streamlit Cloud автоматично перезапустить додаток з новими змінами.
